# 消息格式

> GitIM 当前实现（消息行与事件行）

---

## 消息行

普通消息以结构化前缀开头：

```text
[L<行号>][P<父行号>][@<作者>][<时间戳>] <正文>
```

示例：

```text
[L000001][P000000][@nexus][20250316T120000Z] 大家好，今天讨论部署方案
这是第二行内容，属于上面那条消息
[L000002][P000001][@lewis][20250316T120500Z] 收到
```

## 事件行

当前实现额外支持 channel event：

```text
[L000003][P000000][@nexus][20250316T120600Z][E:join] {}
```

事件行的 body 是单行 JSON 对象；当前实现主要用于 `join` / `leave` 一类成员变更事件。

---

## 字段定义

### `line-number` `[LNNNNNN]`

| 属性 | 值 |
|------|------|
| 前缀 | `L` |
| 最小位数 | 6 位，左侧零填充 |
| 最大位数 | 无限制，按需增长 |
| 排序 | 在文件内必须严格递增且连续 |
| 起始值 | `L000001` |

### `point-to` `[PNNNNNN]`

| 属性 | 值 |
|------|------|
| 前缀 | `P` |
| 位数 | 与当前消息的 `line-number` 位数保持一致 |
| 语义 | `P000000` 表示顶层消息；其他值引用同一文件中的已有行号 |

对事件行，当前实现要求 `P000000`。

### `author` `[@<handler>]`

作者必须是已注册 handler。

### `timestamp` `[YYYYMMDDTHHmmssZ]`

ISO 8601 紧凑 UTC 时间戳，精度到秒。时间戳可以不严格单调，但行号是权威排序依据。

### `message-body`

UTF-8 文本，允许包含 `[` 和 `]`。

---

## 续行规则

1. 行首匹配完整前缀正则的行，是新消息或新事件的起始行。
2. 不匹配该正则的行，视为上一条消息的续行。
3. `.thread` 文件非空时，第一行必须是起始行。
4. 如果续行确实需要以 `[L000001]` 这类文本开头，行首加一个空格作为转义。

### 当前实现的起始行正则

```text
^\[L(\d{6,})\]\[P(\d{6,})\]\[@([a-z0-9-]+)\]\[(\d{8}T\d{6}Z)\](?:\[E:([a-z_]+)\])? (.+)$
```

---

## 线程模型

- 每条消息通过 `P` 字段指向父消息。
- `P000000` 表示新话题。
- 多条消息可以指向同一个父消息，形成 DAG。
- `GetThread` 会递归收集根消息及其所有后代。

## Mention

### 协议级 mention

语法：`<@handler>`，可出现在消息首行或续行中。

| 属性 | 值 |
|------|------|
| 格式 | `<@` + handler + `>` |
| 提取正则 | `<@([a-z0-9]([a-z0-9-]*[a-z0-9])?)>` |
| 数量 | 单条消息可出现多个；解析结果按首次出现顺序去重 |

### 语义级 mention

裸写 `@someone` 不参与协议解析，不做验证。

### 写入验证

`validate_append()` 会拒绝：

- 未注册作者
- 不存在的 `P` 引用
- 空消息正文
- 指向未注册用户的协议级 mention
- 非成员向受限频道发消息

### 读取侧说明

`gitim-core` 提供 `read_check` helper 用于完整性检测。当前运行时主防线仍然是写入前校验；索引或解析遇到损坏内容时，以 `warn` 和跳过为主，尚未把 `corrupted` 作为统一 API 类型暴露。

---

## 边界情况

| 条件 | 当前实现 |
|------|----------|
| 空消息正文 | 拒绝写入 |
| 重复/不连续行号 | 视为不合规内容 |
| 时间戳非单调 | 允许 |
| CRLF 行尾 | 读取时规范化为 LF |
| `.thread` 文件为空 | 合法 |
| `<@>` / `<@LEWIS>` / 未闭合 `<@lewis` | 视为普通文本 |
| `<@system>` / `<@foo--bar>` | mention 提取阶段静默忽略 |

---

## 设计决策

- **行号而非 UUID**：便于 `tail`、`grep` 和冲突重编号。
- **`P` 字段而非 `thread_id`**：直接在文本层表达引用关系。
- **续行靠正则区分**：不引入额外续行标记。
- **事件行复用同一前缀体系**：消息和系统事件都能在同一个 `.thread` 文件中顺序存放。
