## 背景

`WorkspaceUsageHeader`（[products/gitim/frontend/src/components/management/workspace-usage-header.tsx](../../../products/gitim/frontend/src/components/management/workspace-usage-header.tsx)）在 management 页面顶部显示 workspace 的累计 / 今日 token 用量和按 provider 分组的 breakdown：

```
WORKSPACE USAGE
231.2M  累计 · 今日 32.4M · 今日 13 turns
codex 216.7M  opencode 10.1M  claude 4M  pi 447.9K     [sparkline]
```

用户想加一个 toggle 切换 breakdown 分组维度，从 "按 provider" 切到 "按 handler"（每个 agent 单独一项）。

## Scope

**In**：

- breakdown 那一行的分组维度切换：`Provider` ↔ `Handler`
- 顶部数字（231.2M 累计 / 今日 32.4M / 今日 13 turns）和右侧 sparkline 不变
- 状态 per-workspace 持久化到 localStorage（跟 `cardsShowArchived` 同一通道）

**Out**（v1 不做）：

- 多维度交叉（如 by handler 内再按 provider 细分）
- 时间维度 toggle（今日 / 近 7 日 / 近 30 日累计切换）
- breakdown 行点 entry 跳详情 / filter agent 列表

## 共识

### P1 — Toggle 语义

切换只影响 breakdown 行，**不影响**顶部聚合数字和右侧 sparkline（它们本质就是工作区总览，无所谓维度）。

### P2 — 全 0 entry 也显示

- Handler 维度：枚举 workspace **所有** agent，缺 `usageSummary` 的 token 显示为 `0`
- Provider 维度：同步对称改 —— 所有 distinct provider 都列出，无数据贡献为 0
- 排序统一：token 总量降序，0 排尾；同量内按名字字母序 tiebreaker（稳定不抖）

`hasData` 仍然要求至少一个 agent 真有过 usage —— 全 0 的 workspace 整个 header hide（保持现役行为）。

### P3 — 状态持久化

复用项目现有 `lib/ui-state.ts` 通道（已管 `channel` / `boardHandler` / `cardsShowArchived`）：

- `UiState` 加 `usageBreakdown: "provider" | "handler"`，default `"provider"`
- per-workspace key（`gitim-ui-state:<workspaceKey>`）
- 旧 localStorage 数据缺字段 → fallback 到默认 `"provider"`，无需 migration
- 解析校验 narrow：parse 出来不是 `"provider"` 或 `"handler"` → fallback default

### P4 — 控件外观

- 两个并排 `<Button size="sm">`，参考现役 `cardsShowArchived` 风格
- selected 用 `variant="default"` + accent，未选 `variant="ghost"` + muted
- 用 `role="group"` + `aria-label` 包起来，每个按钮 `aria-pressed`
- 文案英文：`Provider` / `Handler`（跟字段名一致，技术术语免去翻译歧义）

### P5 — 布局

Toggle 放 breakdown 行最前，跟 token 列表共享 `flex-wrap`：

```
WORKSPACE USAGE
231.2M  累计 · 今日 32.4M · 今日 13 turns
[ Provider ][ Handler ]   codex 216.7M  opencode 10.1M  claude 4M  pi 447.9K   [sparkline]
   └── 两个独立 Button 并排，selected 高亮
```

- toggle 用 `shrink-0` 不参与 wrap
- token entries 按现役 `flex-wrap` 自然换行
- handler 维度 30 个 agent → 多行展开，header 自然变高（已确认接受）

### P6 — 单一渲染路径

`Provider` 和 `Handler` 共享同一段渲染 JSX，只换数据源：

```ts
const entries = breakdown === "provider"
  ? usage.byProvider.map((e) => ({ key: e.provider, label: e.provider, bucket: e.bucket }))
  : usage.byHandler.map((e) => ({ key: e.handler, label: e.handler, bucket: e.bucket }));
```

避免两份独立 JSX 漂移。

## 验收

- ✅ breakdown 行有可切换的 `Provider` / `Handler` 控件
- ✅ 切换后 token 列表立即换显示，顶部数字与 sparkline 不变
- ✅ 选 Handler、刷新页面：仍是 Handler
- ✅ 切换 workspace：toggle 状态跟随该 workspace 的偏好（不串台）
- ✅ Handler 维度展示所有 agent，无 usage 的显示 `0`
- ✅ Provider 维度同步改造，无数据 provider 也显示 `0`
- ✅ aggregator unit test 覆盖新增 `byHandler` 字段（含 0 case）
- ✅ ui-state.ts 解析校验覆盖（旧字段缺失 / 非法值 → fallback）
- ✅ Header 组件无 a11y regression（`aria-pressed` / `role="group"`）

## 非目标 / 已知约束

- 不动后端 —— runtime `AgentInfo.usage_summary` 已经把每个 agent 用量都暴露给前端，client-side aggregate 即可
- 不动 `byDay` / sparkline 形态 —— sparkline 是时间维度的总用量，不分维度
- 不引入新依赖
