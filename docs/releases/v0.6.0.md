# GitIM v0.6.0

## 新功能

- 协议全面类型化：`/im/*`、`/workspaces`、`/agents`、onboard、search、channel、card、status、send/read 等端点统一使用 typed envelope（Phase 3-4 收尾）
- `MeJson` schema 落地，`me.json` 读写路径统一并支持 merge 语义
- daemon-web：WASM 编译链 + Backend 抽象 + 完整移动端客户端实现（Phase 0-3）
- 前端 browser 模式：支持 cards、card archive、移动端流程、共享 wasm 推断身份、fresh setup 模式选择
- 前端响应式移动端布局（聊天、卡片、侧边栏）
- Hermes ACP prompt response / usage_update 提取 token usage
- pi provider 把 turn_end usage 上报到 compute_snapshot
- runtime tiktoken fallback 估算纳入 tool I/O

## 修复

- `/im/*` 未初始化状态保留 200 状态码
- workspace init 与 daemon readiness 解耦
- 强制 human daemon recovery 后才标记 initialized
- 保留已恢复的 human repo 状态
- pi preflight prompt 输出 flush
- codex 改为解析 `total_token_usage` 而非 rate-limit 百分比
- opencode step usage 解析修复
- system prompt textarea 高度上限,保证按钮可见
- 前端：移动端 toast 不再遮挡 header、cards 导航、runtime 版本检查跳过、browser dev wasm 加载稳定性、移动端 channel cards 入口恢复、移动端 runtime 导航简化

## 改进

- 自托管 release 迁移到 CiferaTeam/GitIM,弃用 gitim-releases
- 开源准备:移除邀请码系统与内部命名
- 产品目录与品牌统一更名为 gitim
- 示例 agent `cifera-nexus` → `code-reviewer`
- README 以 GitIM 优先视角重写,foreground gitim.io 安装路径,补充分布式部署说明
