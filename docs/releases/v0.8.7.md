# GitIM v0.8.7

## 新功能

- **Agent 饱和度采样**：新增 saturation sampler 后台任务，定期采样 agent 工作状态并生成摘要，支持自动清理过期日志
- **Agent 工作状态追踪**：`AgentInfo` 新增 `is_working` 字段 + RAII guard 包裹 provider.execute，精确反映 agent 是否正在处理请求
- **看板跟随用户归档**：归档用户时自动归档其关联的看板

## 修复

- **sync 子进程超时与磁盘满分类**：git 子进程调用增加进程级超时，ENOSPC 错误正确归类
- **sync loop 监控**：daemon 监控 sync loop task 和 auth circuit heartbeat，异常退出自动重启
- **看板归档回收**：修复 list 过滤、rollback 布尔值等问题
- **卡片路由**：thread 更新时正确唤醒卡片的 reporter 和 assignee
- **hard delete 清理**：删除 agent 时同步清理 saturation 日志文件
- **clippy 阻塞项**：修复 gitim-runtime 和 gitim-cli 的 clippy deny 级问题

## 改进

- **git 子进程性能**：将阻塞式 git 子进程调用移出 tokio worker 线程，避免阻塞异步运行时
- **Agent 提示词优化**：CLI-amnesia 提示从 system prompt 移到唤醒时注入；新增引导 agent 不要绕过 gitim CLI 的提示
- **CI 质量门禁**：新增 clippy merge gate workflow，PR 合并前强制通过 lint 检查
