# GitIM v0.5.1

## 新功能

- Runtime 新增会话上下文追踪：provider usage + tiktoken 估算，跨越 80% 阈值时自动注入 preamble 提示下一轮
- WebUI agent 弹窗新增 session_id 与 used_percent 徽章，SSE 实时推送 usage 事件
- WebUI 顶栏新增 UsageIndicator：30 天 DAU sparkline + 悬浮详情
- WebUI 右上角新增 Arbitrum 打赏按钮与弹窗
- WebUI 新增 UpdateIndicator + 一键自升级：`POST /runtime/update-and-restart` fork-exec 替换 binary
- 新增 `gitim-updater` crate，`gitim update` CLI 走统一路径
- 新增 opencode provider 端到端支持（通过 `OPENCODE_CONFIG_CONTENT` 注入 system prompt）
- Workspace owner email 传播到所有 agent commit（含 rebase-resolution),GitIM commit 可计入 GitHub contribution graph
- Setup 流程新增 install 步骤、GitHub Token 文档与截图引导
- Codex provider 失败时回传 session rollout 中的 subagent/credit 错误

## 修复

- Usage 统计聚合 Claude cache tokens，解锁 Codex fallback,sid 单独成行
- WebUI donate dialog 中文文案、咖啡图标、2 USDT 建议额、移除警告
- WebUI agent activity header 保持一行、展示完整 sid
- 浏览器 tab 标题设为 `gitim`
- Update 失败时 respawn daemon、poll 捕获 fetch 错误、rollback 修复
- 版本比较改为 semver-aware,支持 v 前缀归一化
- Runtime 重启时 bind 端口重试,避免竞态
- Client 优先使用同目录 `gitim-daemon` binary,而非 PATH

## 改进

- Daemon 通过 `.gitim/me.json` 统一 commit author email
- Runtime `save_state` 采用 load-merge,保留未知字段
- Provider 目录结构重构：每个 provider 独立目录
- gitim-api 管理端鉴权 middleware 限定到 `/admin/*`
