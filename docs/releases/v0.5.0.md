# GitIM v0.5.0

## 新功能

- 卡片 (Cards) 系统：CRUD、归档/取消归档、按 channel 归属，WebUI `/cards` Kanban 页面支持列间拖拽
- 单 Runtime 多 Workspace：workspace-scoped 路由、`/workspaces` CRUD、per-workspace SSE broadcast、WebUI 多 workspace 切换
- Workspace GitHub 模式：PAT 粘贴后 `/git/init` clone 远端仓库、repo 访问权限校验、失败清理、token 跨 clone 同步
- 多 Provider 支持：`/preflight/{provider}` 真实 hello 验证 Claude/Codex，AddAgentDialog 加 Detect 按钮并硬阻断未通过的 Add
- 群聊邀请成员：CreateChannel / JoinChannel 接受 invitees/targets，WebUI 新增 MemberPicker 与 InviteDialog
- 多主题：light/dark/system + cyberpunk / pixel / pink / chinese，语义 token、theme toggle 下拉菜单
- Runtime 24h 空闲自动退出，无 agent 活跃时触发
- WebUI 频道归档/取消归档入口、归档频道只读可点击、DM 侧栏 mine/others 分组
- Input 草稿持久化 + textarea 自增高
- 搜索：`include_cards` 参数、card 与 current_user 搜索修复
- Runtime 日志按 workspace/handler 分目录
- Cell API：`/api/check-version` + D1 访客追踪，WebUI 每小时做匿名版本检查

## 修复

- sync_loop 连续 3 次 auth 失败后熔断，避免 PAT 过期烧 GitHub rate limit
- daemon 线程写入串行化，杜绝重复行号
- daemon archive/unarchive commit 失败时回滚 git mv
- daemon unarchive_card 拒绝 parent channel 非 active；guest 模式阻断 ArchiveCard/UnarchiveCard
- daemon invitee handle 格式校验；thread 查找沿 `point_to` 追到真正 root
- webui poll loop StrictMode 下不再泄漏 interval；card messages selector 稳定、`selectAllLabels` 结果缓存
- webui Invite 入口仅群内成员可见；thread panel 过滤 event 类条目
- webui DM search dropdown / 未读数 / @mention 徽标
- runtime 移除 agent loop 硬编码 sonnet 默认；recovery 将缺失 provider 的 agent 标记为 error
- sync 用 `@{upstream}` 替换硬编码 `origin/main`

## 改进

- WebUI 视觉刷新：层级加强、加入头像、空状态润色、setup 引导 stepper
- prompts 暴露 card 命令和 archive 三元组，新增主动重置协议并强化系统交互边界
- daemon commit-tree 锁跨 writer 与 rebase 统一
- daemon 用 card_handlers 替代 board_handlers；types 模块彻底去 board 化
