# GitIM v0.8.0

## 新功能

- 新增 Kimi ACP provider，支持 Kimi runtime 流程与本地上下文用量估算
- 新增 Cursor provider（基于 stream-json 信封解析）
- 新增 Provider model catalog，支持各 provider 模型列表的拉取与选择
- 新增 Fleet 模式：SSH tunnel 管理、节点状态追踪、按节点查看 agents、SSE observer
- 新增 Agent routing v1：daemon 在 poll 时附带 recipients，runtime 按 recipients 过滤 prompt，收敛 cascade
- 新增 Provisioning preflight gate：`agents_add` 在 provision 前跑真实 LLM hello 验证，失败零 durable artifact
- 新增 Cards 跟随 channel 归档：archive/unarchive channel 时连带处理 cards 子目录，引入 `archived_via` 字段，daemon 启动时 reconcile 孤儿 cards
- 新增 gitim-runtime CLI 子命令（status / runtime-id / workspaces / list-agents / add-agent / burn-agent / update-agent / preflight），单 binary 双模式
- 新增 pi runtime model catalog 加载
- 新增 Codex live context usage 的 SSE 流式推送
- 新增 self-hosted git CORS proxy

## 修复

- 修复 Kimi 模型 catalog 与用量显示
- 修复 Kimi 在 `set_session_model` 失败时丢失 session_id
- 修复 Cursor runtime 集成、错误信封与流截断升级为 Failed
- 修复 Hermes add-agent E2E 流程
- 修复 opencode preflight 在 git worktree 下的执行
- 修复 Claude result usage fallback 捕获
- 修复 chat header 中 `@` / `#` 前缀重复
- 修复 archive_channel 回滚时 mv 失败未记录、yaml 在 dir-entry 迭代出错时未还原等若干 daemon 一致性问题
- 修复 frontend 在 channel archive poll 后未刷新 cards

## 改进

- Fleet workspaces 按 remote identity 对齐
- 归档频道分页与懒加载
- 浏览器 send / sync 操作串行化，避免竞态
- Workspace usage header 增加 Provider / Handler 维度切换，并按 workspace 持久化
- Per-node usage 详情可折叠
- Runtime CLI：分动作超时（默认 30s，provision 300s）、5xx transient 分类、argv 错误退出码规范化、env secret 脱敏
- Runtime 将 `listen_port` 写入 `runtime.json` 供 CLI 发现
- Hermes preflight 支持 `env_override` 与读取 default profile LLM 配置
- ACP 提取共享 `AcpClient`，Hermes 与 Kimi 复用，并修复 stream-close hang

## 其他

- Rust toolchain 锁定 stable，retire legacy positional CLI 模式
