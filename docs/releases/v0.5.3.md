# GitIM v0.5.3

## 修复

- **claude-opus-4-7 默认 1M context 窗口**，无需显式指定 `[1m]` 后缀即可享受完整窗口
- **修复 context 窗口计量逻辑**：改用每次迭代的实际 token usage 而非累计值，解决窗口核算持续偏高的问题
- **RESET 事件正确清零 HUD session_usage** 并触发 terminal 刷新事件
- **修复 Agent preamble 自相矛盾**：重写约 80% 内容，消除指令冲突导致的行为漂移
- 替换 nightly-only `floor_char_boundary` 为 stable 等价实现，stable toolchain 下不再报错

## 新功能

- **自升级 SHA256 校验（fail-closed）**：`install_update` 在解压前强制校验 SHA256，不匹配直接拒绝，防止二进制损坏或中间人替换
- **结构化升级错误码**：update 错误合约新增 `sha_mismatch` / `sha_line_missing` 两个明确错误类型，供 WebUI 展示区分
- **install.sh 扩展至 4 个平台**（linux-x86_64、linux-aarch64、darwin-x86_64、darwin-aarch64），全部强制 SHA256 验证

## 改进

- Release pipeline 升级为 4 目标矩阵构建，产出 SHA256SUMS 文件
- 仓库锁定 Rust stable toolchain，消除 nightly 依赖，发布产物可复现
- daemon 移除 `openssl-sys` 传递依赖，改用 `rustls-tls`，减少系统库依赖
