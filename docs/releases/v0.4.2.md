# GitIM v0.4.2

## 破坏性变更

- `GET /preflight/claude` 路由移除，改为 `GET /preflight/{provider}` 路径参数形式。调用方需要将 `claude` 作为路径参数传入
- `POST /agents/add` 的 `provider` 字段从可选改为必填；缺省或未知值会返回 400，`AddAgentDialog` 等前端入口必须显式传递 `claude` / `codex`
- 启动时 recovery 逻辑变化：`me.json` 中缺少 `provider` 字段或字段值非 `claude` / `codex` 的 agent 不再静默回退到 claude，而是标记为 `status: error` 并通过 SSE 推送 `event_type=error`；用户需手动修复 `me.json` 才能恢复 agent

## 新功能

- 多 Provider 支持：Add Agent Dialog 可选 Claude / Codex，新增 `GET /preflight/{provider}` 端点跑真实 hello 验证 CLI 可用性
- WebUI 引导页优化，开发模式下跳过邀请码门槛
- WebUI 添加 Cloudflare Pages 部署脚本
- Codex provider 支持通过 AGENTS.md 覆盖 memory/cold_start prompt
- Provider trait 新增 PromptContext 和 prompt 生成方法

## 修复

- WebUI 统一中文文案，区分首次连接与重连的 UX 提示
- cell-api 添加错误处理和中文错误信息
- cell-api 允许 Cloudflare Pages 域名的 CORS 请求
- cell-api 修正 KV binding 名称与 Cloudflare 配置一致

## 改进

- 敏感配置文件（`.env.*`、`wrangler.toml`）从版本控制中移除
