# GitIM v0.5.5

## 新功能
- 新增 Pi provider 支持（含 RPC 模式、preflight 与集成测试）
- 新增 Hermes provider 支持，每个 agent 自动获得独立的 hermes profile（`~/.hermes/profiles/gitim-<handler>/`），LLM 配置、auth、sessions 完全隔离
- Agent 支持 hard delete（彻底清理 agent 目录与 hermes profile）
- Agent model 字段可编辑
- CLI 新增 `leave-channel` 命令；agent prompt 同步暴露离开频道能力
- Codex provider 支持 steering cancel
- 侧边栏会话支持置顶

## 修复
- Hermes ACP 初始化注入 gitim system prompt，避免新 agent 缺少身份上下文
- Hermes API 失败正确上报为 `ExecStatus::Failed`，不再被静默吞掉
- Hermes 增加 authenticate 步骤，修复 resume session_id 异常
- Pi RPC 调用与 CLI protocol 对齐
- 隐藏已下线 agent 的陈旧 DM 频道
- 多 agent 场景下保持聊天频道可见
- Runtime tool 路径支持展开（`~`、env vars）
- Codex runtime 失败正确上报
- 自 leave 不再允许把 meta.members 清空
- GitHub noreply email 在 token propagation 中正确传递
- Hermes 加入 add-provider 白名单与 recovery guard

## 改进
- Codex runtime reasoning effort 默认提升到 xhigh
- 支持 stdin body 输入
- Active agent 预览面板紧凑化
- Agent prompt 警告 daemon 范围 pkill 与共享状态写入风险
- 移除 legacy 代码
